## Frontend Pages Expectations

These instructions apply inside `frontend/pages/`.

## Working Agreements

- Treat page work as user-facing product work.
- Check route behavior, empty states, and mobile responsiveness after changes.
- If a page adds or changes meaningful product behavior, verify whether navigation and docs should also change.
- Prefer page-local fixes first, but audit shared components when the issue looks systemic.

## Common High-Impact Pages

- `home.tsx`
- `map.tsx`
- `adoption-planner.tsx`
- `insights.tsx`
- `liked-swipes.tsx`
- `swipes.tsx`
- `faq.tsx`
