Use this skill for doc-sync work inside `docs/`.

## Workflow

1. Identify the code or infra source of truth first.
2. Update all affected docs, not just the one you opened first.
3. Prefer exact routes, commands, and repo paths.
4. Remove stale product or deployment claims as needed.

Read `references/docs-sync-checklist.md`.
